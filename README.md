Полная инструкция по установке и настройке проекта
Введение
Данный проект представляет собой веб-систему для:

Совместного редактирования таблицы перемещений товаров (аналог Excel).

Управления аудиозаписью на удалённых агентах (ПК с микрофонами).

Администрирования пользователей и просмотра истории изменений.

Система состоит из:

Сервер на Node.js + Express.

База данных MySQL.

Веб-интерфейс с авторизацией, таблицей и панелью аудио.

Агенты – отдельные ПК, на которых запущен recorder_agent.py (Python).

Инструкция рассчитана на Ubuntu Server 20.04/22.04/24.04 (или десктопную версию).
Если у вас другая ОС, некоторые команды могут отличаться.

1. Подготовка сервера (Ubuntu)
1.1. Обновление системы
Откройте терминал и выполните:

bash
sudo apt update && sudo apt upgrade -y
1.2. Установка Node.js
Установите Node.js версии 18 или выше (рекомендуется LTS):

bash
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt install -y nodejs
Проверьте версии:

bash
node -v   # должно быть v18.x.x
npm -v    # должно быть 9.x.x или выше
1.3. Установка MySQL
bash
sudo apt install mysql-server -y
После установки выполните скрипт безопасности:

bash
sudo mysql_secure_installation
Ответьте на вопросы:

Установите пароль для root (запомните его).

Удалить анонимных пользователей? Y

Запретить удалённый вход root? Y (если сервер локальный)

Удалить тестовые базы? Y

Перечитать таблицы привилегий? Y

2. Создание базы данных и пользователя MySQL
Зайдите в MySQL под root:

bash
sudo mysql -u root -p
Введите пароль root, который задали ранее.

Теперь выполните следующие SQL-команды (замените your_password на надёжный пароль):

sql
CREATE DATABASE transfers CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'transfers_user'@'localhost' IDENTIFIED BY 'your_password';
GRANT ALL PRIVILEGES ON transfers.* TO 'transfers_user'@'localhost';
FLUSH PRIVILEGES;
EXIT;
3. Развёртывание проекта
3.1. Создайте папку проекта
bash
mkdir ~/transfers-app
cd ~/transfers-app
3.2. Скопируйте все файлы проекта
Вам нужно перенести все файлы из предыдущих ответов в соответствующие подпапки.
Если у вас есть архив проекта — распакуйте его сюда.

3.3. Настройка конфигурационных файлов
Файл .env
Создайте файл .env в корне проекта:

bash
nano .env
Вставьте:

env
PORT=8000
DB_HOST=localhost
DB_USER=transfers_user
DB_PASSWORD=your_password
DB_NAME=transfers
SESSION_SECRET=my_very_secret_key_change_me
Замените your_password на тот, что указали при создании пользователя MySQL.
SESSION_SECRET придумайте свой сложный набор символов.

Файл config/agents.json
Если вы планируете использовать аудио-агентов, создайте файл config/agents.json:

bash
nano config/agents.json
Пример содержимого (подставьте реальные IP-адреса компьютеров, где запущен агент):

json
[
  {
    "name": "PC-1",
    "ip": "192.168.5.60",
    "port": 5000
  },
  {
    "name": "PC-2",
    "ip": "192.168.1.51",
    "port": 5000
  }
]
4. Инициализация базы данных
4.1. Создание структуры таблиц
Выполните SQL-скрипт sql/init.sql.
Для этого зайдите в MySQL:

bash
mysql -u transfers_user -p transfers
Введите пароль. Затем выполните:

sql
SOURCE ~/transfers-app/sql/init.sql;
Или просто скопируйте содержимое init.sql и вставьте в консоль MySQL.

4.2. Создание первого администратора
В скрипте init.sql уже есть команда:

sql
INSERT INTO users (username, password_hash, roles) VALUES 
('admin', '$2b$10$YourHashedPasswordHere', 'admin,audio');
Но вам нужно заменить '$2b$10$YourHashedPasswordHere' на реальный хеш пароля admin123.

Сгенерируйте хеш с помощью Node.js:

bash
node -e "console.log(require('bcrypt').hashSync('admin123', 10))"
Скопируйте вывод (что-то вроде $2b$10$...).

Теперь обновите запись в базе. Если вы уже выполнили скрипт с заглушкой, исправьте:

sql
UPDATE users SET password_hash = 'скопированный_хеш' WHERE username = 'admin';
Если запись ещё не создана, вставьте правильную команду.

5. Установка зависимостей Node.js
В папке проекта выполните:

bash
npm install
Это установит все необходимые пакеты (express, mysql2, socket.io, bcrypt и др.).

6. Запуск сервера
6.1. Запуск вручную
bash
npm start
Если всё хорошо, вы увидите:

text
Server running on http://localhost:8000
Теперь откройте браузер на другом компьютере в той же сети и перейдите по адресу http://<IP-сервера>:8000.
Если сервер локальный — http://localhost:8000.

6.2. Запуск как службы (чтобы не закрывать терминал)
Установите pm2:

bash
sudo npm install -g pm2
Затем запустите приложение через pm2:

bash
pm2 start server.js --name transfers-app
pm2 save
pm2 startup
После перезагрузки сервера приложение запустится автоматически.

7. Проверка работы
7.1. Вход в систему
Откройте http://<IP-сервера>:8000

Введите логин admin и пароль admin123

Вы должны попасть на главную страницу с таблицей.

7.2. Проверка ролей
В админке (кнопка «Админка») создайте нового пользователя с ролью audio (без admin).

Выйдите и войдите под этим пользователем — вы должны сразу попасть на страницу аудио.

Создайте пользователя без ролей (просто оставьте чекбоксы пустыми) — он должен видеть только таблицу.

8. Настройка агентов аудиозаписи (если нужны)
8.1. Установка Python на компьютеры-агенты
На каждом агенте (ПК, где будет записываться звук) установите Python 3.8+ и необходимые библиотеки:

bash
sudo apt update
sudo apt install python3 python3-pip portaudio19-dev -y
pip3 install sounddevice flask wave
8.2. Создание папки для записей
На агенте создайте папку для хранения записей, например:

bash
mkdir -p ~/audio_records
Или укажите сетевой путь (например, \\Mikola\tds\record — как в вашем примере).
Главное, чтобы у агента был доступ на запись в эту папку.

8.3. Настройка agent_config.json на агенте
На агенте создайте файл agent_config.json рядом со скриптом recorder_agent.py:

json
{
  "base_path": "/home/user/audio_records"
}
Или:

json
{
  "base_path": "\\\\Mikola\\tds\\record"
}
8.4. Запуск агента
bash
python3 recorder_agent.py
Убедитесь, что агент слушает порт 5000:

text
Recorder Agent running on ...
 * Running on http://0.0.0.0:5000
Для постоянной работы используйте systemd или pm2 (для Python есть pm2 с поддержкой Python).

8.5. Проверка связи с сервером
На сервере выполните:

bash
curl http://<IP-агента>:5000/status
Должен прийти JSON с состоянием.

8.6. Добавление агента в конфиг сервера
В файле config/agents.json на сервере укажите IP и порт агента.

9. Устранение неполадок
9.1. Не отображается таблица после входа
Откройте консоль браузера (F12) → вкладка Console. Посмотрите ошибки.

Проверьте, запущен ли сервер (pm2 status или npm start).

Проверьте, правильно ли указан порт в .env и не занят ли он.

9.2. Ошибка подключения к базе данных
Проверьте параметры в .env.

Убедитесь, что MySQL запущен: sudo systemctl status mysql.

Попробуйте подключиться вручную: mysql -u transfers_user -p -h localhost.

9.3. Кнопка «Аудио» не появляется у пользователя с ролью audio
Проверьте, что в базе у пользователя в поле roles есть audio (без пробелов).

Перезайдите заново.

Посмотрите в консоли браузера, приходит ли currentUser.roles в ответе /auth/me.

9.4. Не работает аудио (не воспроизводится)
Убедитесь, что агент доступен с сервера (curl http://<IP-агента>:5000/status).

Проверьте, что файлы существуют на агенте.

Возможно, нужен правильный путь в agent_config.json.

10. Резюме
Поздравляем! Вы установили и настроили полноценную систему совместной работы с таблицами и аудиозаписями.
Теперь вы можете:

Добавлять пользователей и назначать им роли (admin, audio).

Редактировать таблицу в реальном времени с блокировкой ячеек.

Управлять записью звука на удалённых компьютерах и прослушивать файлы.

Просматривать историю изменений таблицы.

Если возникнут вопросы или ошибки — проверьте логи сервера (pm2 logs transfers-app) и логи агентов.

Успешной работы!
